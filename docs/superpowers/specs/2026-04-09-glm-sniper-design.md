# GLM Sniper — 设计文档

**日期：** 2026-04-09  
**目标：** 自动抢购智谱 AI "lite 连接包季" coding plan

---

## 背景

智谱 AI 的 lite 连接包季每天北京时间 10:00 限量开售，手动操作因竞争激烈无法抢到。本程序实现全自动登录 + 精准定时 + 毫秒级抢购，到达"确认订单"页后停止，由用户手动完成付款。

---

## 项目结构

```
glm-sniper/
├── .env              # 账号密码（不提交 git）
├── sniper.py         # 主程序入口
├── config.py         # 配置（目标 URL、抢购时间）
└── requirements.txt
```

---

## 运行流程

```
启动 sniper.py
    ↓
打开 Chrome（有头模式）
    ↓
登录智谱账号（手机号 + 密码）
    ↓
导航到"lite 连接包季"商品页
    ↓
等待至 09:59:50 北京时间（NTP 校时）
    ↓
每 100ms 轮询：
  - 刷新页面
  - 检查购买按钮是否可点击
    ↓
按钮可点 → 立刻点击 → 走完下单流程
    ↓
到达"确认订单"页 → 声音提醒 + 暂停，等用户手动付款
```

---

## 技术细节

### 时间同步
使用 `ntplib` 对齐网络时间，消除本机时钟偏差。

### 轮询策略（9:59:50 起）
1. 刷新页面
2. 查找购买按钮（CSS selector + 文字匹配双保险）
3. 按钮存在且未禁用 → 立刻点击，退出轮询
4. 等待 100ms → 下一轮

### 下单流程
- 点击购买按钮
- 自动选择"季付"选项（如有多档）
- 点击"确认"/"立即购买"
- 检测到"确认订单"页 → 终止自动化

### 凭据管理
```
# .env
GLM_PHONE=<手机号>
GLM_PASSWORD=<密码>
```
`.env` 列入 `.gitignore`，不上传代码仓库。

### 依赖
```
playwright
python-dotenv
ntplib
```

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 登录失败 / 出现验证码 | 立刻停止，打印原因 |
| 按钮 30 秒内未出现 | 停止轮询，保存截图到本地 |
| 网络抖动刷新失败 | 自动重试，不中断主循环 |
| 检测到"售罄"文字 | 立刻停止，终端提示"本次未抢到" |
| Session 失效 | 重新登录后继续 |

## 不在范围内

- 自动付款（用户手动扫码/支付）
- 多账号并发抢购
- 部署到服务器（当前设计为本机运行）
